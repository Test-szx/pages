权限限制似乎只是在提交文件的时候有，而我重启之后，发现并没有权限限制。

在一般情况下，dw 可以删除连续的空格，但并非绝对可以删除，因为这个 w 只是对单词的删除，而不是连续的空格。

GitHub 折腾这种网站很麻烦，尤其是在不熟悉 Git 的时候更甚。

据我观察需要经历以下步骤：

本地文件修改后，找到该文件，并将该文件添加到改动过的分类中。

添加后，提交到本地的 Git 仓库。

提交后，上传到远程仓库。此处需要注意的是，若远程仓库进行了一些读写操作，则本地仓库上传时，必须合并仓库后，才可上传本地仓库修改过的文件。

我当时所经历的步骤：

用安装的 Git 命令行，先切换到仓库的根目录：

cd /盘符/本地仓库路径/
切换到之后，输入 git status 并找到输出中的修改过的未曾添加的文件，并把这些文件逐一添加： 
git add 文件路径
添加后，直接一条命令提交到本地仓库去： 
git commit -m "提交说明"
本地提交完成后，上传到远程仓库： 
git push origin gh-pages
如果你在上传之前，操作过远程仓库的内容，那么你必须先将远程仓库的操作同步到本地仓库，这样才可以成功上传到远程仓库。具体请使用： 
git pull

来完成同步的操作，不过可能会出现问题，具体还得参考Git - 远程仓库的使用来操作，以免后患。
